# ST-GNN Outbreak Prediction System
### Model2 — Production Model with GOA + Two-Factor Verification

---

## What Changed (vs Previous Version)

| Aspect | Old | New |
|--------|-----|-----|
| Data scale | 20 facilities | 240 pharmacies (8 divisions) |
| Data format | `facilities.csv` / `facility_id` | `pharmacies.csv` / `pharmacy_id` (API still accepts `facilities.csv`) |
| Graph edge radius | 10 km | **2.5 km** (tighter, more accurate) |
| Feature vectors | 1 feature (total qty) | **5 features** (per disease group) |
| Hyperparameter tuning | Manual / DGOA inline | **GOA** (Grasshopper Optimization Algorithm — separate step) |
| Training epochs | 100 | **500** (with best-state tracking) |
| ALERT threshold | `model_prob >= 0.85` | **`model_prob >= 0.60`** |
| WATCH threshold | `model_prob >= 0.35` | `model_prob >= 0.35` (unchanged) |
| Two-factor verification | ALERT = sales spike + social signal | Same — but fires earlier (0.60 not 0.85) |
| Social media integration | Optional (no-op if missing) | Optional (generates sample CSV if missing) |
| Dispatcher | MARL multi-agent demo | **Peer-to-peer** greedy nearest-neighbor |
| New output files | — | `data/predictions.csv`, `data/inventory_requests.csv` |

## How to Run

### Full pipeline (steps 1-6)
```bash
python run_all.py
```

Or run each step separately:
```bash
python data/generate_bangladesh_data.py  # Step 1: generate data
python utils/build_graph.py              # Step 2: build graph
python models/optimize_goa.py            # Step 3: GOA hyperparameter search
python models/train_stgnn_pytorch.py     # Step 4: train model
python models/predict.py                 # Step 5: run inference
python models/marl_dispatcher.py         # Step 6: dispatch inventory
```

## Input Data Schemas

### pharmacies.csv (training)
```
pharmacy_id, name, upazila, lat, lon
PH0001, Dhaka Care Pharmacy 1, Dhaka, 23.852128, 90.270003
```

### facilities.csv (API — accepted by pipeline.py)
```
facility_id, name, upazila, lat, lon
PH001, Your Facility Name, Dhaka, 23.810, 90.412
```

> `pipeline.py` accepts **either** `facilities.csv` (with `facility_id`) **or** `pharmacies.csv`
> (with `pharmacy_id`). Output always uses `facility_id`.

### sales.csv
```
date, pharmacy_id (or facility_id), medicine_name, quantity_sold
2024-01-01, PH0001, Paracetamol 500mg, 28
```

### medicines.csv
```
medicine_name, base_daily_sales, outbreak_multiplier, signals_disease
Paracetamol 500mg, 30, 3.5, Fever/Flu
```
`signals_disease` must be one of: `Fever/Flu` | `Diarrhea` | `Respiratory` | `Allergy/Fever` | `Normal`

### social_media_hashtags.csv (optional)
```
date, upazila, hashtag_dengue, hashtag_diarrhea, hashtag_fever, hashtag_respiratory, hashtag_allergy
2024-01-01, Dhaka, 15, 2, 25, 1, 0
```

## Output Schemas

### Pipeline output (`pipeline.py`)
```json
{
  "facility_id": "PH001",
  "upazila": "Dhaka",
  "outbreak_probability": 0.8459,
  "outbreak_flag": true
}
```

### Enriched API output (`app_outbreak` — v1 endpoint)
```json
{
  "facility_id": "PH001",
  "upazila": "Dhaka",
  "model_prob": 0.8459,
  "social_score": 1.0,
  "final_confidence": 0.8459,
  "status": "ALERT",
  "likely_disease": "Fever/Flu",
  "social_confirmation": "(Confirmed by Social Media)",
  "outbreak_probability": 0.8459,
  "outbreak_flag": true
}
```

### Status values
| Status | Condition |
|--------|-----------|
| `ALERT` | `model_prob >= 0.60` AND `social_score >= 0.10` |
| `Normal` | `model_prob >= 0.60` BUT no social signal (false alarm filter) |
| `WATCH` | `0.35 <= model_prob < 0.60` |
| `Normal` | `model_prob < 0.35` |

### Neighbor output (v1 API)
```json
{
  "results": [...],
  "neighbors": {
    "PH001": [
      {"facility_id": "PH021", "upazila": "Dhaka", "distance_km": 2.264}
    ]
  }
}
```

### predictions.csv (standalone training output)
```
pharmacy_id, upazila, model_prob, social_score, final_confidence, status, likely_disease, social_confirmation
PH0001, Dhaka, 0.8459, 1.0, 0.8459, ALERT, Fever/Flu, (Confirmed by Social Media)
```

### inventory_requests.csv (dispatcher output)
```
requesting_pharmacy, requesting_region, supplying_pharmacy, supplying_region, requested_medicine, quantity, distance_km
```

## API Integration (app_outbreak)

The `app_outbreak` service wraps `model2.pipeline` for HTTP inference.

### POST `/run_outbreak` (legacy)
```json
{
  "data_path": "minio://ml-outbreak-input/sales.csv",
  "callback_url": "http://server-a/api/outbreak_callback"
}
```

### POST `/api/v1/ml/jobs/outbreak`
```json
{
  "job_id": "uuid",
  "prediction_horizon_days": 7,
  "input": {
    "snapshot_id": "uuid",
    "sales_file_path": "minio://ml-input/sales.csv",
    "facilities_file_path": "minio://ml-input/facilities.csv"
  },
  "model_version": "outbreak-stgnn-v2",
  "max_neighbors": 20,
  "callback": {
    "url": "https://server-a/api/v1/ml/callbacks/",
    "timeout_seconds": 10
  }
}
```

Response: `202 Accepted` with `{"accepted": true, "job_id": "...", "status": "accepted"}`
